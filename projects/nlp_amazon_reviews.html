Predicting Amazon Review Ratings from Text

This project uses Natural Language Processing (NLP) to classify Amazon product reviews based on written text. The goal was to uncover consumer insight by leveraging text-based features and machine learning models.

GitHub Repository

🗃️ Dataset

Source: Amazon Product Reviews on Kaggle

Size: 500,000+ reviews

  • review_text: Freeform customer review content
  • rating: Target variable (1 to 5 stars)
  • verified_purchase, helpful_vote, category, and other metadata

🧹 Data Preprocessing

  1. Handled missing values in title and review_text
  2. Analyzed rating distributions (found left-skewed pattern)
  3. Text Cleaning:
    • Tokenization and lowercasing
    • Stopword removal
    • Sentiment scoring (TextBlob)
    • Emoji detection
    • Bigrams (CountVectorizer)

🔍 Key Insights

Most Common Phrases in 1-star vs 5-star Reviews

I used CountVectorizer to extract bigrams. Common themes included:

  • 1-star reviews: "poor quality", "stopped working", "waste money"
  • 5-star reviews: "highly recommend", "works great", "love product"

Topic Modeling by Rating

After lemmatization, I used LDA to extract dominant themes per rating:

  • 1-star: useless, return, cheap, broken
  • 5-star: love, great, easy, perfect

Sentiment Analysis vs Star Rating

I compared sentiment polarity with the actual star rating and trained models to predict the rating from review text.

🧠 Modeling Approach

  • Naive Bayes
  • Logistic Regression
  • Random Forest

Evaluation: Accuracy, F1 Score, and Confusion Matrix

Best F1 Score: 0.81 — with room for improvement via hyperparameter tuning or class balancing.

🚀 Future Work

  • Focus on top-reviewed products to derive more actionable insights
  • Handle class imbalance (e.g., down-sample dominant ratings)
  • Incorporate review metadata (e.g., product category, helpful votes)

  • Python: Pandas, NumPy, Scikit-learn
  • NLP: TextBlob, Gensim, CountVectorizer
